# Internationalization (i18n)

Ratic uses [Fluent](https://projectfluent.org/) via the `i18n-embed` + `i18n-embed-fl` + `rust-embed` crates.

## Required Crates

```toml
# Cargo.toml
i18n-embed = { version = "0.16", features = ["desktop-requester", "fluent", "fluent-system"] }
i18n-embed-fl = "0.10"
rust-embed = "8"
```

## `i18n.toml`

Place at the project root (same level as `Cargo.toml`):

```toml
fallback_language = "en-US"

[fluent]
assets_dir = "i18n"
```

## Directory Layout

```
i18n/
├── en-US/
│   └── myapp.ftl       # English (fallback)
└── fr-FR/
    └── myapp.ftl       # French
```

The `.ftl` filename must be consistent across all language directories. By convention it matches the crate name.

## `src/localization.rs` Template

```rust
//! Localization setup.

use std::sync::LazyLock;

use anyhow::Result;
use i18n_embed::fluent::{FluentLanguageLoader, fluent_language_loader};
use i18n_embed::{DefaultLocalizer, DesktopLanguageRequester, LanguageLoader, Localizer};
use rust_embed::RustEmbed;

/// Embeds the contents of the `i18n/` directory into the binary at compile time.
#[derive(RustEmbed)]
#[folder = "i18n/"]
struct Localizations;

/// Global language loader — initialized on first access.
///
/// Uses the system locale (via `$LANG` / `$LANGUAGE` / `$LC_ALL`) with `en-US` as fallback.
pub static LANGUAGE_LOADER: LazyLock<FluentLanguageLoader> = LazyLock::new(|| {
    let loader = fluent_language_loader!();
    let requested = DesktopLanguageRequester::requested_languages();
    loader.load_languages(&Localizations, &requested).expect("Failed to load languages");
    loader
});

/// Returns the crate's localizer (used in `setup()`).
#[must_use]
pub fn localizer() -> Box<dyn Localizer> {
    Box::new(DefaultLocalizer::new(&*LANGUAGE_LOADER, &Localizations))
}

/// Selects the best available language for the current system locale.
///
/// Must be called once in `main()` before any `fl!()` invocation.
///
/// # Errors
///
/// Returns an error if the requested language could not be selected.
pub fn setup() -> Result<()> {
    let localizer = localizer();
    let requested = DesktopLanguageRequester::requested_languages();
    localizer.select(&requested)?;
    Ok(())
}

/// Wrapper around [`i18n_embed_fl::fl!`] that supplies the global loader automatically.
///
/// Usage:
/// ```rust
/// fl!("key")               // → &str
/// fl!("key", var = value)  // → &str with variable substitution
/// ```
#[macro_export]
macro_rules! fl {
    ($message_id:literal) => {
        &i18n_embed_fl::fl!($crate::localization::LANGUAGE_LOADER, $message_id)
    };
    ($message_id:literal, $($args:expr),*) => {
        &i18n_embed_fl::fl!($crate::localization::LANGUAGE_LOADER, $message_id, $($args),*)
    };
}
```

## `.ftl` Syntax

Fluent Translation List files use a simple key-value format:

```fluent
### Section comment (not translated)

## Sub-section comment

# Simple string
app-title = My Application

# String with a variable
greeting = Hello, { $name }!

# Multi-line value (indented continuation)
long-description =
    This is the first line.
    This is the second line.

# Attribute (used for e.g. button tooltips)
save-button = Save
    .tooltip = Save the current file
```

### Complete Example from Ratic

```fluent
### Global

title = Ratic

### Main content

## Navigation

main-menu-about = About
main-menu-button-alt = Main menu
main-menu-settings = Settings

view-albums = Albums
view-artists = Artists
view-library = Library

## Toasts

library-load-end = Finished to load the library
library-check-end = Finished to check the library

### Settings

settings-general = General
settings-default-library = Default library
settings-default-library-subtitle = Folder that will be loaded if none is given at launch.
```

## Usage in Rust

```rust
use crate::fl;

// Simple key:
let title: &str = fl!("app-title");

// With variable (Fluent named-argument syntax):
let msg: &str = fl!("greeting", name = "Alice");
```

`fl!()` returns `&str` (borrowed from the `LazyLock`). Store as `&str` or clone to `String` as needed.

### `LazyLock<String>` for Menu Labels

Menu models are built once and need owned strings. Use `LazyLock<String>` for static labels:

```rust
use std::sync::LazyLock;

static MENU_SETTINGS_LABEL: LazyLock<String> =
    LazyLock::new(|| fl!("main-menu-settings").to_owned());
static MENU_ABOUT_LABEL: LazyLock<String> =
    LazyLock::new(|| fl!("main-menu-about").to_owned());

// Then in the `menu!` macro:
menu! {
    main_menu: {
        section! {
            &MENU_SETTINGS_LABEL => SettingsAction,
            &MENU_ABOUT_LABEL    => AboutAction,
        }
    }
}
```

## Extracting / Updating Translation Strings

```bash
cargo i18n    # Extracts new keys from Rust source, updates all .ftl files
```

This requires the `cargo-i18n` binary (`cargo install cargo-i18n`). New keys are added to all language files with empty values; untranslated keys fall back to `en-US`.
