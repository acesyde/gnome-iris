# Project Structure

## Canonical Directory Layout

```
my-app/
├── build.rs                    # Compiles icons, GResources, GSettings schemas
├── Cargo.toml
├── Cargo.lock
├── rust-toolchain.toml
├── rustfmt.toml
├── i18n.toml
├── data/
│   ├── icons/
│   │   ├── icons.gresource.xml # GResource manifest for custom SVG icons
│   │   └── hicolor/scalable/apps/
│   │       └── logo.svg
│   └── org.example.myapp.gschema.xml
├── i18n/
│   ├── en-US/
│   │   └── myapp.ftl
│   └── fr-FR/
│       └── myapp.ftl
├── packages/                   # Nix packaging (optional)
└── src/
    ├── main.rs
    ├── localization.rs
    ├── patterns.rs              # Optional: fuzzy search helpers
    ├── ui/
    │   ├── mod.rs
    │   ├── window.rs            # Root component
    │   ├── header.rs            # Orchestrator / ViewStack host
    │   ├── about.rs
    │   ├── settings.rs
    │   └── <feature>.rs         # One file per UI component
    └── <domain>/                # Backend logic, zero GTK imports
        ├── mod.rs
        ├── cache.rs
        ├── player.rs
        └── <model>.rs
```

## Module Conventions

- Every `src/<dir>/` has a `mod.rs` that starts with a `//!` module doc comment and declares `pub mod` for each submodule.
- The domain layer (`src/<domain>/`) must not import GTK crates — keep it pure Rust.
- `src/ui/` may import from `src/<domain>/`, never the reverse.
- Clippy denies `missing_docs`: every `pub` item needs `///` and every module file needs `//!`.

### Example `src/<domain>/mod.rs`

```rust
//! Core domain logic for my-app.

pub mod cache;
pub mod model;
pub mod player;
```

### Example `src/ui/mod.rs`

```rust
//! User interface of my-app.

use std::sync::LazyLock;

use relm4::gtk::gdk::Texture;

pub mod about;
pub mod header;
pub mod settings;
pub mod window;

/// GResource path for the application logo SVG.
pub const APPLICATION_LOGO_RESOURCE: &str =
    "/org/example/myapp/icons/scalabale/actions/hicolor/scalable/apps/logo.svg";

/// Decoded texture of the application logo (loaded once at first access).
pub static APPLICATION_LOGO: LazyLock<Texture> =
    LazyLock::new(|| Texture::from_resource(APPLICATION_LOGO_RESOURCE));
```

## `src/main.rs` Template

```rust
//! # My App
//!
//! Brief description of the application.

// Nightly features (add only what you actually use):
#![feature(never_type)]

use std::io::Write;
use std::sync::Arc;

use anyhow::Result;

// Icon names generated by relm4_icons_build in build.rs.
#[allow(missing_docs)]
#[allow(clippy::doc_markdown)]
pub mod icon_names {
    pub use shipped::*;
    include!(concat!(env!("OUT_DIR"), "/icon_names.rs"));
}

pub mod localization;
pub mod <domain>;
pub mod ui;

use directories::ProjectDirs;
use relm4::gtk::{gdk, gio};
use relm4::{RELM_THREADS, RelmApp, gtk};
use tokio::sync::RwLock;

use crate::<domain>::cache::{Cache, ImageCache};
use crate::ui::window::Window;

/// D-Bus application ID. Must match the GSettings schema id and the GResource prefix.
pub const APPLICATION_ID: &str = "org.example.myapp";

/// Returns the XDG directories for this application.
#[must_use]
pub fn application_dirs() -> Option<ProjectDirs> {
    ProjectDirs::from("org", "example", "myapp")
}

/// Registers the compiled GResource bundle and adds the icon theme path.
fn initialize_custom_resources() {
    gio::resources_register_include!("icons.gresources").unwrap();
    let display = gdk::Display::default().unwrap();
    let theme = gtk::IconTheme::for_display(&display);
    theme.add_resource_path("/org/example/myapp");
}

fn main() -> Result<()> {
    // 1. Icons from relm4-icons + custom GResources.
    relm4_icons::initialize_icons(icon_names::GRESOURCE_BYTES, icon_names::RESOURCE_PREFIX);
    initialize_custom_resources();

    // 2. Structured logger (level controlled by RUST_LOG env var).
    env_logger::builder()
        .format(|fmt, record| writeln!(fmt, "{}: {}", record.level(), record.args()))
        .init();

    // 3. GStreamer (omit if the app has no audio).
    gst::init()?;

    // 4. Global CSS overrides.
    relm4::set_global_css(
        "
            .sidebar-pane { background: rgba(0, 0, 0, 0); }
        ",
    );

    // 5. i18n.
    localization::setup()?;

    // 6. Worker thread pool size (tune to app needs).
    RELM_THREADS.set(4).expect("Could not set thread count");

    // 7. Shared caches initialised before the UI starts.
    let cache = Arc::new(RwLock::new(Cache::init()));
    let image_cache = Arc::new(RwLock::new(ImageCache::new()));

    // 8. Launch the root component.
    let app = RelmApp::new(APPLICATION_ID);
    app.run::<Window>((cache, image_cache));

    Ok(())
}
```

### Notes on `main.rs`

- `APPLICATION_ID` must be the same string used in the GSettings schema `id` and the GResource `prefix`.
- `RELM_THREADS` sets the Tokio thread-pool size for async components. Ratic uses 5.
- Caches are created with `Arc::new(RwLock::new(...))` and passed as `Init` data into the root component, which then distributes them to children.
- `localization::setup()` must be called before any `fl!()` macro invocation.
- GStreamer init (`gst::init()`) is only needed if the app plays audio.
